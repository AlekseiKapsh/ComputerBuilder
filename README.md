1. Computer (Продукт)
Основной объект, который создается
Содержит характеристики: процессор, видеокарта, память, хранилище и т.д.
Метод DisplayConfiguration() выводит все параметры

2. IComputerBuilder (Интерфейс строителя)
Определяет шаги сборки компьютера
8 методов для установки каждой части компьютера
Метод GetResult() возвращает готовый продукт

3. Конкретные строители (3 типа):
GamingComputerBuilder - мощная игровая конфигурация
OfficeComputerBuilder - бюджетная офисная конфигурация
BudgetGamingComputerBuilder - средний вариант для игр

4. ComputerDirector (Директор)
Управляет процессом сборки
Предлагает готовые "рецепты":
BuildFullComputer() - полная сборка
BuildBasicComputer() - только основные компоненты
BuildMinimalComputer() - минимальная конфигурация

Что происходит в Main():
Сборка с директором:
Игровой компьютер - самая мощная конфигурация с жидкостным охлаждением
Офисный компьютер - экономичная версия со стандартным охлаждением
Бюджетный игровой - баланс цены и производительности

Ручная сборка (без директора):
Демонстрация гибкости - можно вызывать методы строителя в любом порядке
Можно пропускать некоторые шаги

Ключевые особенности паттерна:
Разделение ответственности - логика сборки отделена от самого продукта
Гибкость - можно создавать разные вариации одного объекта
Контролируемая сборка - директор обеспечивает правильный порядок шагов
Расширяемость - легко добавить новый тип компьютера (например, серверный)
